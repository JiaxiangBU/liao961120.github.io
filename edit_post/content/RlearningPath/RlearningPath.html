接觸 R 的時間大約五個月了，從原本對電腦、程式一竅不通到現在能有效率的 debug 、寫出簡潔有條理的 R code 、甚至用 R 與 Markdown 架站寫部落格。算一算，我每週通常至少 3 天會用到 R ，不是督促自己熟悉 R ，是因為它太有魅力了。我現在只要有作業沒有要求使用Word，幾乎都會用 Rmarkdown 或 Markdown 去完成，再輸出成 PDF 或 html ，因為總忍不住要碰一下 R 。

學 R 語言很有趣1，但也相當耗時耗力。我在學習 R 上花了大量的時間，若不是因緣際會找到對的方向，我不可能花費那麼多力氣去學 R ，甚至可能直接放棄。雖然如此，我還是感到有點可惜，若能更早知道有效率的學習方式，我就不必花費如此大量的時間(以及大量的紙張)。因此，我希望將學習 R 的心路歷程寫下，說明為何我可以持續學習而未放棄，以及若從頭來過，我會挑選哪些學習資源，讓學習可以更有效率。

為何我能持續學習 R 而未放棄？其實現在我比較頭大的問題是：要怎麼克制自己不要一直打開 Rstudio ？正是因為我學 R 的過程如同打電動一樣歡樂，所以放棄這個問題根本不存在。事實上，剛開始學 R 時並不怎麼歡樂，直到上了學校的課(見下課程回顧)才越來越喜歡 R 。

課程回顧： R 語言與資料科學導論

106學年上學期，我最喜歡、也意外收穫最多的課是謝舒凱老師開設的 R 語言與資料科學導論，或許會是我大學生涯最喜歡的課吧。

起初是抱著學好 R 語言(加上通識快修不完)的心情去加簽這門課的，但課堂上 R 語言的語法卻教得不多。然正是這種不聚焦在程式語法的課程安排，讓我把 R 學得非常好。如果課程從頭到尾都在教 R 語言，到期末我一定會受不了越顯複雜的語法2，最後便隨意敷衍了事。我在這門課收穫最大的，反而不是課堂學到的技巧，而是老師、助教們傳達的一些概念與想法，以及自己探索這些概念想法的樂趣與收穫：

  • R的生態圈3以及 Open Source 的概念

  • 資料科學與 Story telling: Coding 於資料科學中的角色不同於其在工程上的角色，亦及 Aesthetic Coding 與 Functional Coding 的概念。資料科學不僅包含傳統的 Functional Coding，也包含大量的 Aesthetic Coding，用於傳達資料科學上的發現

  • Reproducibility: Rmarkdown (.Rmd) 是我見過最強大的文件格式，可以輸出成網頁、投影片、PDF、Word、Markdown 等。這讓使用者可以在一份文件中做事情(Rmd)，並依據所需輸出成各種文件，而不用剪剪貼貼(從Word貼到 PPT)，也省下排版花費的心力。例如，要報告可輸出成投影片、要列印可輸出成PDF、要放到網路上可輸出成網頁。這讓使用者能有效率的管理與記錄工作流程，是達到 Reproducibility 的基礎。

  • 文本分析: 我本來對這部分最沒有興趣，直到我意識到文本資料是研究人類行為、社會與文化最重要的資料來源之一4,5

課程起手式

這門課與我個性相符，完全顯現在第一堂課實習課(Lab Session)的作業上。作業與R並無相關，而是要使用Markdown格式撰寫自我介紹。由於我對簡約風有點痴迷6 (受 Markdown 影響後更為痴迷)，我便在 Markdown 上花了一些時間。目前，透過R與 Pandoc Markdown 的整合，我可以用 Rmarkdown 簡潔快速地寫出含有上下標、Footnotes、超連結、citation等等的文件7 (不須使用html語法)，我甚至未來考慮使用Rmarkdown撰寫論文。

對Rmarkdown的痴迷與熱愛可說是我R語言功力進步的關鍵。Rmarkdown賦予每個人將自己的想法傳達給其他人的能力8，因此，能將美好的事物以及想法與大家分享成為我學習 R 最強烈的動機。例如，當初在做這門課的期末專案時，發覺 R 可以畫出各種互動式圖表，於是我開始尋找最適合呈現資料的方式。由於不同種類的圖，有時需要不同格式的資料9，因此為了畫出最適合的圖，我反覆整理許多資料以符合格式要求，使得我 data-wrangling 的能力大幅提升。
